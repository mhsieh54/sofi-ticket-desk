# Handoff — 2026-07-02

## Session 4: Dashboard build

### What was built
- Next.js 16 app in `dashboard/`, App Router, TypeScript.
- **Auth**: password-gated via `proxy.ts` (Next 16's renamed middleware convention) + HMAC-signed session cookie (Web Crypto, works in both Node and Edge runtimes).
- **Reads**: `GET /api/positions` reads `dashboard/data/*.json`, a local copy synced from the project-root `data/` by `scripts/sync-data.js` (runs via `predev`/`prebuild`). Direct cross-root reads (`../data`) don't work — Turbopack refuses to trace paths outside the project root and hard-crashes the build (confirmed via testing, not a guess).
- **Writes**: `PATCH /api/positions` commits changes back to `data/inventory.json` / `data/sold.json` on GitHub via the Contents API (`lib/github.ts`), so every phone/browser edit becomes a real git commit. "Mark sold" moves a position from inventory to sold and computes `soldPayout` using the same platform fee table as CLAUDE.md.
- Single-page dashboard UI: stats (capital in, realized/projected P&L, category counts), positions grouped by event with urgency labels, inline Edit and Mark Sold forms, collapsible sold history.

### Decisions made (with Mark)
- Cloud hosting on Vercel (free tier), not Tailscale — Mark has no always-on machine, so a private-network approach wasn't practical.
- Interactive editing required (not read-only) — Mark wants to mark tickets sold / adjust prices from his phone, e.g. at the stadium.
- Writes persist via committing to GitHub (not a separate database) — keeps git as the single source of truth and preserves the audit-trail model the project already uses.
- Password-gate auth, since writes are reachable from the open internet once deployed.

### Bugs caught and fixed during testing (not just written and assumed correct)
- **Turbopack cross-root path crash**: originally planned to read `../data` directly with an `outputFileTracingIncludes` config; `next build` panicked with `glob '../data/**' is invalid, it has a prefix that navigates out of the project root`. Fixed by copying `data/*.json` into `dashboard/data/` via a sync script instead (see above).
- **Type error in `lib/fees.ts`**: `(platform && PLATFORM_FEES[platform]?.seller) ?? 0` didn't type-check cleanly (TS couldn't narrow past the `platform` fallback in `&&`). Fixed to `platform ? PLATFORM_FEES[platform]?.seller ?? 0 : 0`.
- **`middleware.ts` deprecated in Next 16** — build warned it's been renamed to `proxy.ts`. Migrated: file renamed, exported function renamed `middleware` → `proxy`, confirmed via web search against the official Next.js migration notes.
- **Dependency versions**: initial `next@14.2.18` resolved to a patch version with an unpatched high-severity advisory (DoS via Server Components). Bumped to `next@16.2.10` (current stable) + `react@19`. One moderate postcss advisory remains (build-tool-only, not exploitable here — the suggested "fix" would downgrade Next.js to 9.x, which is wrong).

### What was actually tested (live, in a running dev server)
- Login flow end-to-end (password → session cookie → dashboard renders).
- Logout (cookie cleared, redirected to `/login`).
- `GET /api/positions` — real data loads correctly; spot-checked the math (Realized P&L $1,133 verified by hand against all 7 sold positions' individual profit, including a -$1.80 Monster Jam breakeven loss that's easy to miss manually).
- `PATCH /api/positions` (mark sold) — confirmed it fails cleanly with a clear error banner (`GITHUB_OWNER is not set`) when the GitHub env vars aren't configured yet, and that the failed attempt doesn't corrupt or lose any local state.
- Note: the browser-automation tool's simulated clicks didn't reliably trigger React's event handlers in this environment (confirmed via direct comparison — a native `.click()` call worked every time, the tool's own click didn't for buttons wired to async handlers). This looks like a testing-tool quirk, not an app bug, but flagging it since it means "the tool clicked it and nothing happened" isn't proof of a real bug — always cross-check with a native click or network log before concluding something's broken.
