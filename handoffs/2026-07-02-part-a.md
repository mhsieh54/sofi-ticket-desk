# Handoff — 2026-07-02 (Part A: scheduled market agent)

## What was built
Part A of the comp-checker ("C + A together" — Part C, the manual Comps tab, shipped earlier today in `fbe53a4`).

- **`scripts/market-brief.mjs`** — a standalone Node/ESM script. Reads `data/inventory.json`, groups active SELL positions by event (7 events currently: Karol G, BTS, R&B Sep 26, and Rams Packers/Chiefs/Cowboys/Seahawks-sell), and asks Claude (with the web-search server tool) to write a dated market brief to `data/briefs/YYYY-MM-DD.md`. Brief format: bottom-line summary → per-event read + recommendation → watch list. Encodes the desk's rules in the system prompt (fees, 30% ROI floor, BTS $625 ceiling, 2nd-3rd-cheapest targeting).
- **`scripts/package.json`** — isolates `@anthropic-ai/sdk` from the dashboard so it never touches the Next.js build.
- **`.github/workflows/market-brief.yml`** — runs the script Mon/Wed/Fri at 14:00 UTC (~6-7am LA) and on-demand (`workflow_dispatch`). Commits the brief with the default `GITHUB_TOKEN` (has `contents: write`); the push triggers Vercel's redeploy so the brief propagates.

## Key decisions
- **GitHub Actions + Claude API**, not a scheduled cloud routine (Mark's pick) — fully self-contained in the repo, free (GitHub cron + minutes), transparent.
- **Model defaults to `claude-opus-4-8`** per the claude-api skill's guidance (don't downgrade for cost without the user's say-so). Made it overridable: the workflow's `model` input and a `MODEL` env var both flow through, so Mark can switch to `claude-sonnet-5` for lower cost without a code change. Flagged the cost tradeoff in the README.
- **Narrative, not floors** — the brief gives directional market read (the honest limit of web-search-based pricing); exact section floors stay in `data/comps.json` via the manual Comps tab. Documented this distinction in the brief header and README so the two aren't confused.
